### [1] [OptiMer: Optimal Distribution Vector Merging Is Better than Data Mixing for Continual Pre-Training](https://arxiv.org/abs/2603.28858)
*Haiyue Song,Masao Utiyama*

Main category: cs.CL

TL;DR: The paper introduces OptiMer, a method that turns the choice of continual pre-training (CPT) data mixture ratios into a post-hoc optimization problem using distribution vectors from separately trained models, achieving better performance at much lower search cost.


<details>
  <summary>Details</summary>
Motivation: In continual pre-training of large language models, selecting the right mixture ratio of different training datasets (e.g., languages, domains) is crucial but costly to tune. Ratios must be fixed before training starts, and a bad choice can waste significant compute over weeks of training. The authors aim to avoid this rigid, expensive trial-and-error process by enabling flexible, efficient adaptation to languages and domains after CPT has been done.

Method: They propose OptiMer, a two-stage framework. First, they perform continual pre-training separately on each dataset (e.g., Japanese, Chinese, Math, Code), yielding one CPT model per dataset. For each model, they compute a distribution vector that captures the parameter shift caused by that dataset relative to the base model. Second, they perform post-hoc Bayesian optimization over the composition weights of these distribution vectors to find an optimal linear combination for a target objective, effectively constructing a target model without retraining. The optimized weights can also be interpreted as implied data mixture ratios, which can be used to retrain a standard data-mixture CPT model if desired.

Result: On Gemma 3 27B, across languages (Japanese, Chinese) and domains (Math, Code), OptiMer consistently outperforms standard data mixture CPT and model averaging baselines, while reducing the search cost by 15–35×. The learned composition weights are meaningful as mixture ratios: using them to retrain data-mixture CPT further improves performance. Additionally, once a pool of distribution vectors is trained, the same pool can be re-optimized for new target metrics or tasks without any additional CPT, allowing on-demand generation of specialized models.

Conclusion: The paper demonstrates that data mixture ratio selection for continual pre-training, historically a static hyperparameter choice made before training, can be reframed as a post-hoc optimization problem over distribution vectors derived from per-dataset CPT models. This reformulation enables more flexible, efficient, and interpretable adaptation of LLMs to new languages and domains, delivering better performance with substantially lower search cost and enabling rapid re-targeting without retraining.

Abstract: Continual pre-training is widely used to adapt LLMs to target languages and domains, yet the mixture ratio of training data remains a sensitive hyperparameter that is expensive to tune: they must be fixed before training begins, and a suboptimal choice can waste weeks of compute. In this work, we propose OptiMer, which decouples ratio selection from training: we train one CPT model per dataset, extract each model's distribution vector, which represents the parameter shift induced by that dataset, and search for optimal composition weights post-hoc via Bayesian optimization. Experiments on Gemma 3 27B across languages (Japanese, Chinese) and domains (Math, Code) show that OptiMer consistently outperforms data mixture and model averaging baselines with 15-35 times lower search cost. Key findings reveal that 1) the optimized weights can be interpreted as data mixture ratios, and retraining with these ratios improves data mixture CPT, and 2) the same vector pool can be re-optimized for a given objective without any retraining, producing target-tailored models on demand. Our work establishes that data mixture ratio selection, traditionally a pre-training decision, can be reformulated as a post-hoc optimization over distribution vectors, offering a more flexible paradigm for continual pre-training.
